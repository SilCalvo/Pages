AMR - LASER MAPPING
Silvia Calvo Cabello

    • INTRODUCCIÓN

    La práctica consiste en generar un mapa del entorno a partir de la información obtenida por el sensor LIDAR del robot. El objetivo principal es desarrollar un algoritmo de navegación capaz de explorar de forma autónoma un área de almacén mientras construye un mapa probabilistico preciso del entorno utilizando los datos del láser. También se realizará la comprativa de este mismo algoritmo mediante empeorando la localizacion del robot, para ver los efecto en la genereracion del mapa.

    • MAPA

    Para construir el mapa a partir de los datos del láser es necesario conocer varios parámetros fundamentales:

    • El sensor proporciona 360 lecturas, una por cada grado.
    • El láser 0 apunta hacia delante y el láser 270 hacia la derecha.
    • El alcance máximo del sensor es de 3.5 metros.
    • Es imprescindible conocer la orientación del robot para poder transformar correctamente los ángulos del láser al marco global.
    Con esta información se trazan los rayos del LIDAR sobre el mapa, se realiza un mapa probabilistico, en caso de detectar libre aumenta la probabilidad de libre un (RELLENAR) y en caso de detectar ocupado baja la probabilidad de estar libre un (RELLENAR).

    Para garantizar la independencia entre observaciones en la construcción del mapa probabilístico, se ha aplicado un muestreo angular del sensor láser. En lugar de utilizar todos los rayos, se selecciona únicamente uno de cada 5 (salto = 5), lo que introduce una separación angular suficiente para reducir la correlación entre medidas consecutivas. Esto permite aproximar la hipótesis de independencia entre observaciones requerida por los métodos de mapeo probabilístico, manteniendo al mismo tiempo una cobertura adecuada del entorno.

    • RECORRIDO

    Para obtener un mapa completo y fiable es necesario diseñar un recorrido adecuado. La solución adoptada se inspira en el comportamiento de robots domésticos como las aspiradoras tipo Roomba: primero recorren el perímetro del área para definir los límites del espacio y luego realizan un barrido sistemático del interior. En esta práctica, el robot sigue una estrategia similar:

    • Primero localiza la pared más cercana en la dirección Este.
    • Después recorre todo el perímetro del entorno siguiendo la pared y actualizando el mapa.
    • Una vez delimitado el contorno, se ejecuta un algoritmo basado en BSA que busca la celda libre más cercana de modo que va recorriendo una mayor zona.
    De esta forma el robot obtendrá un mapa bastante completo.

    • CODIGO

    Explicación del programa

    Este programa implementa un sistema autónomo de exploración del entorno mediante mapeo probabilístico con sensor láser usando log-odds y ray tracing (Bresenham), y finalmente la generación de un recorrido tipo cobertura que permite visitar todas las celdas libres del entorno. El sistema combina dos estrategias:

    • Exploración global mediante seguimiento de paredes
    • Exploración interna mediante búsqueda de celdas libres (BFS)
    Utiliza HAL para el control del robot y WebGUI para la visualización, construyendo dinámicamente un mapa del entorno a partir del láser.

    Este sistema funciona con:

    • Mapeo dinámico con mapping()
    • Recorrido de paredes para delimitar el entorno
    • Conversión implícita a celdas mediante discretización
    • Exploración completa usando vecinos + BFS
    • Movimiento preciso con go_to()


    Flujo principal del programa

    El programa sigue los siguientes pasos:

    • Inicialización de mapas y parámetros
    • Llamada a go_init_position():
      • Orienta el robot hacia el Este
      • Avanza hasta encontrar una pared
      • Se orienta hacia el Norte
    • Ejecución de recorrer_pared():
      • El robot sigue la pared usando wall-following
      • Va construyendo el mapa con mapping()
      • Marca las celdas visitadas
      • Finaliza cuando vuelve cerca del punto inicial
    • Ejecución de recorrer_interior():
      • Busca celdas vecinas no visitadas (next_cell())
      • Si no hay, usa BFS (buscar_celda_libre_mas_cercana())
      • Convierte la celda a píxeles
      • Se mueve con go_to()
      • Marca la celda como visitada
      • Repite hasta cubrir todo el mapa
    • Finalización cuando no quedan celdas accesibles

    Al finalizar, el robot habrá mapeado completamente el entorno y recorrido todas las zonas navegables de forma autónoma.


    Funciones del programa

    bresenham()

    Implementa el algoritmo de Bresenham para trazar líneas discretas:

    • Recibe dos puntos (inicio y fin)
    • Devuelve todos los píxeles intermedios
    • Se usa para simular el recorrido del rayo láser
    Es fundamental en el proceso de mapping.

    pixel_a_celda() y celda_a_pixel()

    Permiten convertir entre dos sistemas:

    • Píxeles (mapa continuo)
    • Celdas (mapa discreto)
    • pixel_a_celda(): divide por el tamaño de celda
    • celda_a_pixel(): devuelve el centro de la celda

    avanzar()

    Hace avanzar el robot una distancia determinada:

    • Guarda la posición inicial
    • Se mueve hacia delante
    • Comprueba continuamente el láser frontal
    • Se detiene si:
      • Detecta obstáculo
      • Alcanza la distancia objetivo
    • Durante el avance:
      • Actualiza el mapa con mapping()
      • Marca celdas visitadas

    map_visit_rellenar()

    Marca la celda actual como visitada:

    • Obtiene la pose del robot
    • Convierte a píxeles y luego a celda
    • Marca la matriz mapa_visit con 255
    • Pinta la celda en el mapa

    orientar()

    Orienta el robot hacia un ángulo objetivo:

    • Calcula la diferencia angular
    • Normaliza el ángulo entre [-π, π]
    • Gira en la dirección más corta
    • Se detiene cuando el error es pequeño

    recorrer_pared()

    Implementa seguimiento de pared:

    • Comprueba el lado derecho con el láser
    • Si está libre → gira a la derecha
    • Si está ocupado → avanza recto
    • Si hay obstáculo frontal → gira a la izquierda
    • Actualiza mapa y celdas visitadas
    • Termina al volver al punto inicial

    go_init_position()

    Coloca el robot en una posición inicial adecuada:

    • Se orienta hacia el Este
    • Avanza hasta detectar una pared
    • Se orienta hacia el Norte

    es_obstaculo()

    Determina si una celda es un obstáculo:

    • Convierte la celda a región de píxeles
    • Calcula el valor medio
    • Si es oscuro → obstáculo
    • Si es claro → libre

    buscar_celda_libre_mas_cercana()

    Busca la celda libre no visitada más cercana mediante BFS:

    • Parte desde la celda actual
    • Explora vecinos (N, E, S, O)
    • Evita obstáculos
    • Devuelve la primera celda válida encontrada

    next_cell()

    Devuelve una celda vecina no visitada:

    • Prioridad: Norte → Este → Sur → Oeste
    • Si existe, la devuelve
    • Si no, devuelve None

    go_to()

    Mueve el robot a una posición objetivo:

    • Calcula dirección hacia el objetivo
    • Orienta el robot
    • Avanza hacia el punto
    • Se detiene si:
      • Llega al objetivo
      • Detecta obstáculo
    • Si hay obstáculo:
      • Marca la celda como ocupada
      • Retrocede ligeramente

    pintar_celda_rojo()

    Pinta una celda en el mapa visual:

    • Convierte la celda a píxeles
    • Colorea la región
    • Actualiza la GUI

    recorrer_interior()

    Explora todo el interior del mapa:

    • Busca siguiente celda con next_cell()
    • Si no hay, usa BFS
    • Convierte a píxeles
    • Se mueve con go_to()
    • Marca la celda como visitada
    • Repite hasta terminar

    mapping()

    Construye el mapa de ocupación:

    • Lee datos del láser
    • Convierte cada rayo a coordenadas globales
    • Traza el rayo con Bresenham
    • Actualiza:
      • Zonas libres (L_FREE)
      • Obstáculos (L_OBS)
    • Aplica saturación
    • Convierte log-odds a imagen (0–255)
